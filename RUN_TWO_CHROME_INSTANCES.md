# Chrome에서 Flutter 앱 두 개 인스턴스 동시 실행

채팅·멀티유저 등을 테스트할 때, 같은 앱을 **서로 다른 포트**로 두 번 띄워 두 개의 Chrome 창에서 동시에 시뮬레이션할 수 있습니다.

## 방법 1: 터미널 두 개에서 수동 실행

**터미널 1 (인스턴스 1):**
```bash
flutter run -d chrome --web-port=8080
```

**터미널 2 (인스턴스 2):**
```bash
flutter run -d chrome --web-port=8081
```

- 각 터미널에서 빌드가 끝나면 Chrome 창이 하나씩 열립니다.
- `localhost:8080` / `localhost:8081` 로 서로 다른 세션(쿠키, 로컬 스토리지 등)이 사용되므로, 두 개의 사용자처럼 로그인·채팅 테스트가 가능합니다.

## 방법 2: 프로젝트 루트의 스크립트 사용 (Windows)

프로젝트 루트에 있는 배치 파일을 **두 개의 터미널**에서 각각 실행합니다.

- **터미널 1:** `run_chrome_8080.bat`
- **터미널 2:** `run_chrome_8081.bat`

또는 Cursor/VS Code에서 터미널을 2개 열고 각각 위 배치 파일을 실행해도 됩니다.

## 참고

- **디버깅:** 각 Chrome 창은 각자의 DevTools를 가질 수 있습니다. F12로 각 창에서 따로 디버깅 가능합니다.
- **포트 변경:** `8080`, `8081` 대신 사용하지 않는 포트(예: `9090`, `9091`)로 바꿔도 됩니다.
- **다른 브라우저:** `-d chrome` 대신 `-d edge` 등으로 변경하면 해당 브라우저로 실행됩니다.
